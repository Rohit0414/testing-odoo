How it works

1) Upload & Index

PDFs are split into chunks; each chunk is embedded and upserted into Qdrant with stable UUIDs (doc/page/chunk).

2) Ask a Question

We embed the query, retrieve Top-K vectors, keep best N, and build a token-aware context.

3) Grounded Answer

OpenAI responds strictly from the provided context. If not found, it replies ‚ÄúI don‚Äôt know.‚Äù

4) Citations

Each answer shows the PDF title, page number, and a short snippet for verification.

Install & Configure

Requirements
  • Odoo 18 (community or enterprise)
  • Python deps: qdrant_client, openai, pypdf, tiktoken
  • Qdrant Cloud cluster & API key
  • OpenAI API key
Quickstart
# 1) Copy addon into your Odoo custom_addons/
# 2) Install dependencies in the Odoo venv:
pip install qdrant_client openai pypdf tiktoken

# 3) Update apps list & install from Apps (9-dots) as "Niyu PDF RAG"
# 4) In Settings ‚Üí PDF RAG: paste OpenAI key, Qdrant URL, and Qdrant API key, then Save.
# 5) Open PDF RAG ‚Üí Documents: upload PDF and click "Index Now".
# 6) Go to "Chat" and ask away.
Reverse proxy note (file uploads)
# /etc/nginx/sites-available/odoo
client_max_body_size 100m;
proxy_read_timeout   600s;
proxy_send_timeout   600s;

# Test + reload
sudo nginx -t && sudo systemctl reload nginx

FAQ

Why do I sometimes get ‚ÄúI don‚Äôt know‚Äù?

If retrieval returns no meaningful text (e.g., scanned PDFs without OCR), the assistant answers honestly. Reindex with OCR or verify your Qdrant collection.

Can I tune tokens and Top-K?

Yes ‚Äî in Settings, adjust Top-K, Keep Top, and Answer max tokens. Lower values reduce cost and latency.

How are points identified in Qdrant?

Stable UUIDv5 generated from doc_id-page-chunk so re-indexing overwrites prior chunks for the same location.

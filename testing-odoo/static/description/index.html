Power BI + BigQuery: Production-Ready Setup

Build a clean star schema, keep refresh blazing fast, and publish with confidence. Below is a proven path used on live Odoo workloads.

  1. Get Data ‚Üí BigQuery in Power BI Desktop. Choose OAuth (user login) or Service Account (centralized). Pick your Billing Project and Dataset.
  2. Select filtered tables exported from Odoo: orders, order_lines, partners. Avoid ‚ÄúSELECT *‚Äù.
  3. Model as a star: orders (fact) ‚Üí order_lines (detail) via order_id, and dimensions like partners, products, date.
  4. Define DAX measures (not calculated columns) for performance. Example:
    Total Sales = SUM(orders[amount_total])
    Orders Count = COUNTROWS(orders)
    Sales Last 30D = CALCULATE([Total Sales], DATESINPERIOD('date'[date], MAX('date'[date]), -30, DAY))
  5. Enable Incremental Refresh (Power BI Pro/Premium): partition by order_date. Keep last 2‚Äì3 years + refresh last 7‚Äì14 days.
  6. Publish to Power BI Service ‚Üí set credentials to BigQuery. Schedule refresh based on your BigQuery export cadence.
Pro Tips
  • Use DirectQuery only for small/interactive models; otherwise prefer Import with Incremental Refresh.
  • Mark your calendar table as Date table to unlock proper time intelligence.
  • In relationships, set single direction unless absolutely needed to cross-filter both ways.

Tableau + BigQuery: Visual Speed with Control

Use BigQuery for scale and Tableau for storytelling. Keep datasets slim and push heavy lifting into BigQuery.

  1. Connect ‚Üí Google BigQuery. Choose Standard SQL. Sign in or use a service account JSON.
  2. Choose Live vs Extract. Live for near real-time; Extract for heavy dashboards and offline publish.
  3. Relationships (not joins) first: relate orders ‚Üî order_lines by order_id, and add dimensions like partners, date.
  4. Filters: add a Data Source Filter (e.g., last 365 days) to cap query size before visuals.
  5. Publish to Tableau Server/Cloud and set BigQuery credentials + refresh cadence.
Design Hints
  • Favor relationships for flexibility; switch to physical joins only when necessary.
  • Use LOD expressions for reusable metrics (e.g., customer-level first/last order).
  • Limit high-cardinality dimensions in a single viz; break into detail drilldowns.

Frequently Asked Questions

What do I need to use this module?

An Odoo instance, a Google Cloud account with BigQuery enabled, and a Service Account with "BigQuery Data Editor" and "Job User" roles.

Can it sync data from custom Odoo modules?

Yes. The connector can access and export data from any standard or custom model within your Odoo database, including any custom fields.

Does the sync overwrite the BigQuery table, or can I append data instead? What happens to existing records in BigQuery?

By default, yes. The connector uses WRITE_TRUNCATE to replace the table on each sync, ensuring it's a perfect replica of the source Odoo data. This means all existing records in the BigQuery table are deleted and replaced with the latest export. If you need append or merge functionality, please contact us to discuss your requirements.

Is my data secure during the sync process?

Yes. All data transfers use official Google Cloud authentication and encrypted connections. No third-party servers are involved‚Äîyour data flows directly from Odoo to your BigQuery project.

Can I schedule automatic exports?

Yes. You can schedule recurring syncs directly from Odoo, ensuring your BigQuery tables are always up to date with the latest filtered data.

Does it support large datasets?

Yes. The connector is designed to handle large Odoo tables and will efficiently export even hundreds of thousands of records to BigQuery.

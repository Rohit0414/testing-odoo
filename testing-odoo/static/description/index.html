Important Points to Note

  • 
    Google Cloud Service Account: You must create a Service Account in your Google Cloud Platform project and grant it the "BigQuery Data Editor" and "BigQuery Job User" roles. You will then download its JSON key to use in Odoo.
  • 
    Data Privacy and Security: Your Odoo data is transmitted directly from your server to the Google Cloud API. It is NEVER stored on any third-party server. The connection is exclusively between your Odoo instance and your GCP project.
  • ️
    Correct Configuration is Key: You are in full control. Ensure your Service Account has the correct IAM permissions and that your Project ID, Dataset ID, and JSON key are accurately entered in Odoo for a successful connection.

Frequently Asked Questions

What do I need to use this module?

An Odoo instance, a Google Cloud Platform account with BigQuery enabled, and the ability to create a Service Account with the required permissions.

Can it sync data from custom Odoo modules?

Yes. The connector can access and export data from any standard or custom model within your Odoo database, including custom fields.

Does the sync overwrite the BigQuery table?

By default, yes. The connector uses the `WRITE_TRUNCATE` disposition, which replaces the table on each sync to ensure data consistency. This is the standard for maintaining a replica of Odoo data.

What if the connection fails?

The most common issues are an invalid Project ID, an incorrectly copied JSON key, or insufficient IAM permissions for the Service Account. Double-check that it has "BigQuery Data Editor" and "BigQuery Job User" roles.
